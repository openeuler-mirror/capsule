<script>
    function scaleContentToFit() {
        const sections = document.querySelectorAll('.section-container');
        sections.forEach(section => {
            const contentWrapper = section.querySelector('.scalable-content');
            const content = contentWrapper ? contentWrapper.firstElementChild : null;
            if (!contentWrapper || !content) return;

            contentWrapper.style.transform = 'scale(1)';
            contentWrapper.style.zoom = 1;

            const styles = window.getComputedStyle(section);

            const paddingTop = parseFloat(styles.paddingTop);
            const paddingBottom = parseFloat(styles.paddingBottom);
            const paddingLeft = parseFloat(styles.paddingLeft);
            const paddingRight = parseFloat(styles.paddingRight);

            const ORIGINAL_HEIGHT = section.clientHeight - paddingTop - paddingBottom;
            const ORIGINAL_WIDTH = section.clientWidth - paddingLeft - paddingRight;
            if (Math.abs(contentWrapper.scrollWidth - ORIGINAL_WIDTH) < 1 && Math.abs(contentWrapper.scrollHeight - ORIGINAL_HEIGHT) < 1) {
                window.final_ratio = 1;
                return;
            }

            const TARGET_ASPECT_RATIO = ORIGINAL_HEIGHT / ORIGINAL_WIDTH;

            let testWidth = ORIGINAL_WIDTH;

            let bestWidth = ORIGINAL_WIDTH;
            let minDifference = Infinity;

            for (let i = 0; i < 300; i++) {
                testWidth = testWidth + 5;
                content.style.width = `${testWidth}px`;

                const actualHeight = content.scrollHeight;
                const targetHeight = testWidth * TARGET_ASPECT_RATIO;

                const currentDifference = Math.abs(actualHeight - targetHeight);

                if (currentDifference < minDifference) {
                    minDifference = currentDifference;
                    bestWidth = testWidth;
                }
            }

            content.style.width = `${bestWidth}px`;

            const finalWidth = content.scrollWidth;
            const finalHeight = content.scrollHeight;

            const scaleX = ORIGINAL_WIDTH / finalWidth;
            const scaleY = ORIGINAL_HEIGHT / finalHeight;

            const scaleRatio = Math.min(scaleX, scaleY);
            window.final_ratio = scaleRatio;
            contentWrapper.style.transform = 'none';
            contentWrapper.style.zoom = scaleRatio;
        });
    }

    window.addEventListener('load', scaleContentToFit);
    window.addEventListener('resize', scaleContentToFit);
</script>

#### **1. 脚本简介：它是一个“试探性”布局修正脚本**

这个脚本的核心功能不是简单的等比缩放。它的工作方式更像一个“智能整理师”：

*   **第一步：探索性放大 (Exploratory Expansion)**
    它会通过一个循环，逐渐增加内容区域的总宽度。在每一步放大中，它会实时测量内容的总高度。

*   **第二步：寻找最佳平衡点 (Finding the Optimal Balance)**
    它的目标是找到一个宽度和高度的“理想”组合，使得内容能够最舒展、最自然地排布。它认为，当内容的测量高度与基于宽高比计算出的理论高度非常接近时，就找到了这个平衡点。

*   **第三步：精确缩放至容器 (Precise Scaling)**
    一旦找到这个“理想尺寸”，它就会计算出一个精确的缩放比例（`zoom`），将这个已经“整理好”的内容完美地缩放到父容器中。

#### **2. 关键协作原则：保持高度的“可预测性”**

这个脚本成功的关键，在于它能够**预测**增加宽度后，高度会如何变化。如果内容的CSS导致高度以一种**不可预测**或**非线性**的方式变化，脚本的计算就会失败。

因此，您在生成HTML和CSS时，必须遵循以下核心规则：

**【规则1：绝对禁止使用“宽高比锁定”的CSS技巧】**

**严禁**在任何元素上使用 `padding-top` 或 `padding-bottom` 的百分比值来维持宽高比。

*   **错误示例 (会导致脚本崩溃):**
    ```css
    .image-container {
        /* 这个技巧会让高度随宽度变化，脚本无法处理 */
        padding-top: 56.25%; /* 16:9 ratio */
        height: 0;
        position: relative;
    }
    ```

*   **为什么会冲突?**
    这个技巧会创建一个“宽度->高度”的直接依赖关系。当脚本增加宽度时，您设置的 `padding-top` 会导致元素高度也随之增加。这形成了一个脚本无法解决的循环依赖，导致最终计算出错误的、极小的缩放比例。

*   **正确替代方案:**
    请使用 `flexbox` 或 `grid` 布局来管理尺寸。对于图片，可以直接设置其 `width: 100%` 和 `height: 100%`，或者让其在flex容器中自适应。

**【规则2：优先使用Flexbox进行空间分配】**

请大量使用 `display: flex` 和 `flex: 1` 属性来让元素自动填充可用空间。

*   **推荐示例 (与脚本完美兼容):**
    ```css
    .container {
        display: flex;
        flex-direction: column;
        height: 100%;
    }
    .card {
        flex: 1; /* 自动均分父容器的高度 */
        min-height: 0; /* 防止内容溢出 */
    }
    ```
*   **为什么这样好?**
    Flexbox布局的高度分配是稳定且可预测的。当脚本改变总宽度时，flex项的高度分配逻辑保持不变，不会产生意外的高度增长。这为脚本的计算提供了稳定的环境。

**【规则3：避免使用固定的`height`值（除非必要）】**

尽量避免给主内容区的模块设置固定的像素高度（如 `height: 300px`）。让它们的高度由Flexbox或其内部内容自然决定。这给了脚本在“试探”过程中更大的灵活性。

该脚本是一个强大的布局优化工具，但它依赖于一个**稳定**且**可预测**的CSS环境。您的任务是提供这样的环境。
